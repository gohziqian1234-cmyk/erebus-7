# Project: Erebus-7: First Skin

## The Problem

Many games today are very big, competitive, and hard to learn. Some players do not always want a game that takes many hours or needs a lot of practice before it becomes fun.

For this project, I wanted to create a browser game that feels more unique and memorable. Instead of making a normal action game, I wanted to make a single-player horror game where the player has to think, hide, and make decisions carefully.

The target users are mainly students, young adults, and players who enjoy horror, stealth, and story-based games. The game is also made for people who like games such as Among Us, but want to play alone instead of online with other players.

The challenge was to make the game scary and interesting, while still making sure the player understands what to do. I also wanted the game to feel more polished than a simple prototype.

## What I Built

I built a single-player social-horror stealth game called **Erebus-7: First Skin**.

In the game, the player is an alien parasite trapped inside a space station. The player controls a human host and must infect crew members, avoid being detected, and slowly take over the station.

The main goal is to move through different areas of the station and reach SELENE, the station AI. To win, the player must complete objectives, use infected hosts as helpers, avoid guards and scanners, and survive until the final AI Core chapter.

The game includes movement, enemy patrols, detection systems, infection mechanics, clone respawn, difficulty modes, story scenes, sound effects, tactical map, tutorial, save/load, and multiple chapters.

## Key Features

- Single-player horror stealth gameplay
- Story set on a space station called Erebus-7
- Player controls an alien parasite wearing a human body
- Crew members can be infected and used as helpers
- Detection bar that increases when guards or cameras see the player
- Easy, Medium, and Hard difficulty modes
- Clone system that lets the player respawn at a planted backup body
- Tactical map showing rooms, people, danger, objectives, and routes
- Different station zones such as Crew Quarters, Med Bay, Security Hub, Command Deck, and AI Core
- Enemy types such as guards, hunters, listeners, stalkers, sweepers, and SELENE
- Story scenes and blackbox logs to explain what happened on the station
- Sound design with alarms, footsteps, whispers, scanner sounds, and horror music
- Pause menu, settings, save/load, tutorial, and game-over screens
- GitHub Pages publishing so the game can be played online

## Technologies Used

I used **HTML, CSS, JavaScript, and HTML5 Canvas** to build this project.

HTML was used to create the structure of the game page. This includes the canvas, intro screen, tutorial, HUD, pause menu, tactical map, story scenes, and game-over screens.

CSS was used to design the look and feel of the game. I used a dark horror style with teal, red, green, and blue colours to match the space station and parasite theme.

JavaScript was used to make the game work. It controls the player movement, enemy AI, detection bar, infection system, abilities, map system, story flow, sound, save/load, and game rules.

HTML5 Canvas was used to draw the actual game world. It renders the map, rooms, characters, vision cones, effects, lighting, particles, objectives, and animations in real time.

## Challenges and How I Overcame Them

One challenge was making the game feel scary, not just dark. At first, the game felt too basic because the player was only moving around a map. To improve this, I added stronger room art, horror effects, sound design, story scenes, and scary events when the player enters different areas.

Another challenge was balancing the detection system. If the detection filled too slowly, the game became too easy. If it filled too fast, the game felt unfair. To solve this, I added difficulty modes and adjusted the detection range, detection speed, and cooldown based on the mode.

The enemy AI was also difficult to make. Simple enemies were not scary because they only walked around. I improved this by adding smarter patrols, different enemy types, route cutting, sound investigation, and enemies that react to the player’s actions.

Another challenge was making the player understand what to do. Since the game has many systems, it could be confusing. I improved this by adding a clearer tutorial, mission panel, tactical map, objective marker, and hints during gameplay.

Audio was also a big challenge because browser games usually block sound until the player clicks something. To fix this, I added an Audio On button and made the game unlock sound properly when the player interacts with the page.

## Outcome

The final result is a playable browser horror game with a full story theme, multiple levels, enemies, objectives, sound, and a clear mission flow.

This project helped me understand how much work goes into making a game feel good. I learned that gameplay is not only about controls and rules. A good game also needs feedback, sound, pacing, story, balance, and clear instructions for the player.

Overall, **Erebus-7: First Skin** became much more than a simple prototype. It turned into a complete browser game that can be played online and shared with other people.
